# HK Watch Prices

A local price database built from WhatsApp exports of a Hong Kong watch dealer group. Parses dealer stock lists, deduplicates, and lets you query by reference, color, year, condition, seller, and date.

## What it does

- Reads WhatsApp `_chat.txt` exports placed in `exports/`
- Extracts one row per watch listing using a brand-aware regex parser (~96% capture on observed data)
- Stores everything in `data/watches.db` (SQLite)
- Deduplicates against existing rows on every refresh
- Optional Claude API fallback recovers the ~4% the regex misses
- Query via CLI, CSV export, or browser

## Setup

```bash
cd ~/Projects/hk-watch-prices

# Web UI only — the parser and CLI need no install
pip install -r requirements.txt
```

## Weekly refresh workflow

Each week:

1. In WhatsApp, open the HK 55 Watch club group → **Export Chat → Without Media**
2. Save the resulting `_chat.txt` to `exports/` with a dated name, e.g.:
   ```
   exports/2026-05-28_export.txt
   ```
3. Run:
   ```bash
   python src/refresh.py
   ```

The script:
- Parses every `.txt` it hasn't already loaded (tracked in the `exports_loaded` table)
- Inserts new rows; rows already present (same `posted_at + seller + raw_line`) are skipped
- Prints overall counts so you can sanity-check

To reparse a file you've already loaded (e.g. after a parser change), use `--force`.

### LLM fallback (optional)

To recover the lines the regex parser couldn't handle:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
python src/refresh.py --llm
```

Uses Claude Haiku 4.5 in batches of 25 lines. Expect ~$0.01–0.05 per weekly export.

## Querying

### CLI

```bash
# Single reference, all conditions/years
python src/query.py --ref 5167R

# Median/min/max for matching rows
python src/query.py --ref 5167R --summary

# Combined filters
python src/query.py --brand "Patek Philippe" --year 2022 --condition new

# Date range
python src/query.py --ref 26240OR --since 2026-05-15

# Database stats
python src/query.py --stats

# Most-listed references
python src/query.py --top 20
```

### CSV export

```bash
# Everything
python src/export_csv.py

# Filtered
python src/export_csv.py --brand "Audemars Piguet" --year 2024 --out ap-2024.csv
```

### Web UI

```bash
streamlit run src/web.py
```

Opens a browser with sidebar filters, summary metrics, a price-by-year scatter plot, and downloadable filtered CSV.

## Schema

The `listings` table:

| column | type | notes |
| --- | --- | --- |
| posted_at | TEXT | ISO datetime of the WhatsApp message |
| seller | TEXT | Sender's WhatsApp display name |
| brand | TEXT | Inferred — `Patek Philippe`, `Rolex`, etc., or NULL |
| reference | TEXT | Watch reference (e.g. `5167R`, `126234`, `4520V/210A-B128`) |
| dial_color | TEXT | First color word found (`blue`, `salmon`, …) |
| year_made | INTEGER | 4-digit year, e.g. 2024 |
| month_made | INTEGER | 1–12, only when listing specifies month (e.g. `N5/26`) |
| condition | TEXT | `new` or `used` |
| price_hkd | INTEGER | HKD price |
| price_usdt | INTEGER | USDT price |
| full_set | INTEGER | 1 = full set, 0 = naked / watch-only, NULL otherwise |
| raw_line | TEXT | Original line from the export (kept for verification) |
| raw_message | TEXT | First 500 chars of the parent message |
| source_file | TEXT | Which export the row came from |
| confidence | TEXT | `regex` or `llm` |

Unique constraint on `(posted_at, seller, raw_line)` — protects against double-loading the same export.

## Notes & caveats

- **Brand inference** is heuristic: it uses reference shape first, then context emojis (👑=PP, 🛡️=VC, 🛰️=AP) from the surrounding message. Roughly 1% of rows have `brand = NULL` — usually one-off lines without enough context.
- **Year/month parsing** handles `2024`, `2024y`, `2024/12`, `25/4` (Apr 2025), `N5/26` (new, May 2026). When a line has only a 4-digit year and no month, `month_made` is NULL.
- **Prices**: HKD is the default currency when none is specified (these are HK price lists). USDT is recorded separately when present. `K`/`M`/`mil` suffixes are expanded.
- **WTB messages** ("looking for…") are filtered out — only stock-offering messages are kept.
- The first export covers only the days since you joined the group. Future weekly refreshes will accumulate history.
